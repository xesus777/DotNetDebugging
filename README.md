## Идея проекта

В рамках практической работы выполнена отладка консольного приложения с использованием встроенных средств Microsoft Visual Studio. Целью работы является приобретение навыков использования отладчика для анализа состояния программы, отслеживания ошибок и управления потоком выполнения.

## Использованные средства отладки

| Средство отладки | Назначение |
|------------------|------------|
| **Запуск отладчика (F5)** | Запуск приложения с подключенным отладчиком для пошагового анализа кода и отслеживания состояния переменных |
| **Точки останова (Breakpoints)** | Приостановка выполнения программы на указанной строке кода для анализа значений переменных |
| **Подсказки по данным (Data Tips)** | Быстрый просмотр значений переменных при наведении курсора мыши во время остановки выполнения |
| **Шаг с обходом (Step Over - F10)** | Пошаговое выполнение кода без входа в вызываемые методы |
| **Шаг с заходом (Step Into - F11)** | Пошаговое выполнение с входом в вызываемые методы для детального анализа их работы |
| **Шаг с выходом (Step Out - Shift+F11)** | Выход из текущего метода с возвратом к вызывающему коду |
| **Выполнить до щелчка (Run to Click)** | Быстрое перемещение отладчика к выбранной строке кода без установки постоянной точки останова |
| **Перезапуск приложения (Ctrl+Shift+F5)** | Остановка текущего сеанса отладки и запуск нового с подключенным отладчиком |
| **Окно "Авто" (Autos)** | Автоматическое отображение переменных, используемых на текущей и предыдущей строках кода |
| **Окно "Локальные" (Locals)** | Отображение всех переменных, определенных в текущей области видимости (функции или методе) |
| **Окно контрольных значений (Watch)** | Отслеживание выбранных переменных или выражений в процессе пошагового выполнения кода |
| **Окно стека вызовов (Call Stack)** | Просмотр последовательности вызовов методов и функций для анализа потока выполнения приложения |
