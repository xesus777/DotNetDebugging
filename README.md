## Идея проекта

В рамках практической работы выполнена отладка трех консольных приложений с использованием встроенных средств Microsoft Visual Studio. Целью работы является приобретение навыков использования отладчика для анализа состояния программы, отслеживания ошибок и управления потоком выполнения.

### Использованные средства отладки

| Средство отладки | Назначение |
|------------------|------------|
| Точки останова (Breakpoints) | Приостановка выполнения на определенных строках кода |
| Условные точки останова | Остановка только при выполнении заданного условия |
| Шаг с заходом (Step Into) | Пошаговое выполнение с входом в вызываемые методы |
| Шаг с обходом (Step Over) | Пошаговое выполнение без входа в методы |
| Шаг с выходом (Step Out) | Выход из текущего метода |
| Окно контрольных значений (Watch) | Отслеживание значений выбранных переменных |
| Окно локальных переменных (Locals) | Автоматический просмотр переменных в текущей области |
| Окно стека вызовов (Call Stack) | Просмотр последовательности вызовов методов |
| Немедленное окно (Immediate Window) | Вычисление выражений и изменение значений во время отладки |
| Подсказки по данным (Data Tips) | Быстрый просмотр значений при наведении мыши |
| Окно вывода (Output) | Просмотр точек трассировки и отладочной информации |

## Задание 1. Отладка приложения «Фибоначчи»

### Описание задания
Выполнена отладка консольного приложения, вычисляющего числа Фибоначчи. В процессе отладки исследовано поведение программы при различных входных данных.

### Использованные способы отладки

1. **Точки останова** — установлены на строке вычисления очередного числа для анализа значений переменных.
2. **Шаг с заходом (Step Into)** — выполнено последовательное выполнение рекурсивного метода.
3. **Окно локальных переменных (Locals)** — отслеживание значений `n`, `a`, `b` на каждой итерации.
4. **Подсказки по данным** — наведение мыши для быстрого просмотра значений.
5. **Условная точка останова** — остановка выполнения при `n == 10`.

### Результаты отладки
- Выявлена корректность работы алгоритма.

## Задание 2. Отладка приложения «Галактики»

### Описание задания
Выполнена отладка консольного приложения, выводящего информацию о галактиках.

### Использованные способы отладки

1. **Стек вызовов (Call Stack)** — анализ последовательности вызовов методов при возникновении исключения.
2. **Окно контрольных значений (Watch)** — отслеживание состояния объектов коллекции.
3. **Немедленное окно (Immediate Window)** — выполнение выражений для проверки значений во время отладки.
4. **Точки трассировки (Tracepoints)** — вывод информации в окно вывода без остановки выполнения.
5. **Шаг с обходом (Step Over)** — пропуск выполнения методов, не требующих детального анализа.

### Результаты отладки
- Обнаружена и исправлена ошибка.
- Проверена корректность работы объектов.

## Задание 3. Отладка приложения «Буквы»

### Описание задания
Выполнена отладка консольного приложения, обрабатывающего строковые данные.

### Использованные способы отладки

1. **Временные точки останова (Temporary Breakpoints)** — автоматическое удаление после первого срабатывания.
2. **Окно «Авто» (Autos)** — просмотр переменных, используемых в текущей и предыдущей строках.
3. **Шаг с выходом (Step Out)** — быстрый выход из метода после проверки начальной логики.
4. **Изменение значения переменной во время отладки** — двойной щелчок в окне просмотра для изменения значения.
5. **Параметры отладчика** — настройка поведения отладчика через меню `Tools > Options > Debugging`.

### Результаты отладки
- Проверена корректность обработки.
